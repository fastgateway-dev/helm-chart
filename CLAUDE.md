## Project Overview

Helm chart for **FastGateway** - a Kubernetes Gateway API management platform built on Envoy Gateway. The chart deploys two components: a Go backend (port 8081) and a Next.js frontend (port 3000, service port 3001), backed by an external PostgreSQL database.

## Commands

```bash
# Generate README.md from helm-docs annotations in values.yaml
make readme

# Package the chart and update the Helm repo index
make helm.create.releases

# Lint the chart
helm lint fastgateway/

# Template the chart locally (requires setting required values)
helm template fastgateway fastgateway/ \
  --set database.host=localhost \
  --set database.existingSecret=db-secret \
  --set secrets.existingSecret=app-secret
```

## Documentation

README.md is auto-generated by [helm-docs](https://github.com/norwoodj/helm-docs). The template file is `fastgateway/README.md.gotmpl`. To regenerate, run `make readme`. Do not edit `fastgateway/README.md` directly.

Use `helm-docs` comment annotations in `values.yaml` (e.g., `# -- Description`) to document parameters.

## Architecture

- **Single chart** at `fastgateway/` with two deployments (backend + frontend)
- Backend receives config via ConfigMap (LOG_LEVEL, ADMIN_USERNAME, ADMIN_EMAIL) and Secrets (JWT, encryption key, admin password, DB credentials)
- Frontend connects to backend via internal service URL (`http://<release>-backend:8081`)
- Database credentials and app secrets are provided via `existingSecret` references (not managed by the chart for DB; optionally auto-generated for app secrets)
- RBAC: ClusterRole + ClusterRoleBinding granting access to Gateway API resources, Envoy Gateway CRDs, and core K8s resources
- Optional HPA and PDB per component (backend/frontend), controlled by `*.autoscaling.enabled` and `*.podDisruptionBudget.enabled`
- WAF config supports coraza-proxy-wasm OCI image with optional SHA256 pinning

## Key Conventions

- Template helpers are in `fastgateway/templates/_helpers.tpl` - use existing named templates (e.g., `fastgateway.fullname`, `fastgateway.backend.labels`) for consistency
- Deployments include `checksum/secret` and `checksum/config` annotations to trigger rolling updates on config changes
- Required values (`database.host`, `database.existingSecret`) use Helm's `required` function and will fail at template time if missing
- Chart version is in `fastgateway/Chart.yaml`; packaged releases go to `releases/`
